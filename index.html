Intro to Programming Overview

Lesson 1

Thinking Like a Programmer

There are 5 main types of thinking that are frequently used by programmers:
  1. Procedural
    Involves creating clear and concise instructions for a computer to follow.
  2. Abstract
    Involves finding ways to improve programming efficiency by helping avoid
    unnecessary repetition of work.
  3. Systems
    Involves breaking down large problems into smaller problems and recognizing
    how those individual pieces fit together.
  4. Technological Empathy
    Involves understanding how computers work and what they are good
    and bad at and using that knowledge to your advantage while coding.
  5. Debugging
    Involves using a system to make identifying and fixing bugs quick
    and relatively easy.

The Basics of the Web and HTML

The World Wide Web
A network of servers and other computers on which
HTML documents and files are hosted and accessed.
Components of the WWW
-Users who use computers to access and create content on the WWW.
-Computers which run programs (browsers) that display pages and files found
on the web.
-The Internet which is the largest network of computers in the world.
-Servers which are computers that host the files that make up the web.
-HTTP which is the protocol that is used to communicate between these
components. This Wikipedia entry explains HyperText Transfer Protocol in
extreme depth.
Webpages
Text documents, written in HTML, which are read, interpreted, and displayed
by web browsers.
HTML
HyperText Markup Language is a programming language which is the basis of
almost every webpage on the WWW. HTML documents consist of:
-Text content (what you see)
-Markup (how it looks)
-References to other documents/files (images, video, audio, other pages, etc.
(This video is an excellent introduction to HTML.)
Components of HTML
-Tags: The text instructions that tell a browser how to display content.
They are always contained within angled braces. They're paired together with
an opening and closing tag, except for void tags.
-Elements: A unit made up of a tag and the content it affects, or an opening and
closing tag and the content between them.
Inline vs Block Tags
The two types of tags within HTML are inline and block tags. Block tags form
a box around the contents wihin them.
Coding
The action of writing text instructions in a language that computers can
understand and follow.
Correct Syntax
Computers are "stupid" because they require code to be written
in a very exact way to be able to understand it. Humans are considered "smart," by
contrast, because they are capable of compensating for and overcoming typos.

Lesson 2

Creating a Structured Document with HTML

Text Editors
The programs used to write and edit the code for webpages. They are similar to
word processors like Microsoft Word, but lack the formatting functions. Good
text editors have many automatic and predictive functions that make programming
easier. They can often be used to write code in multiple languages.
HTML vs CSS vs JavaScript
-HTML is the language used to define the structure and content of a webpage.
-CSS is the language used to define the style of the webpage.
-JavaScript is the language used to define the interactive components of a webpage.
DOM
Document Object Model is the standard convention used in representing
and interacting with elements in HTML. This means that browsers all render
HTML using the same model, resulting in a common pattern in the structure of
all webpages (see Branching Structure below).
Browser Developer Tools
Add-ons or built-in features that allow you to view the DOM trees that browsers
interpret HTML documents as, and the styles applied to the elements contained with
these HTML documents.
Branching Structure
Because HTML elements can be nested within each other, the structure of an HTML
document can be analogized to a tree. The more deeply nested within other elements
an individual element is, the further up and out from the trunk of the tree it would
be found. For example, an image might be nested within a 'div' element which is, in
turn, nested within the 'body' element. The body may be considered the trunk of the
tree, the div, a bough sticking out from the trunk, and the image, a twig sticking out
from the bough.
Nested Boxes and Indentation
Every single visible element of a webpage is bounded by a box and, therefore,
every webpage can be broken down into a series of boxes with smaller boxes inside
of them (and often more boxes inside of these smaller boxes, and so on). Front end
web developers do this with design mockups of webpages to begin laying out the
framework of a page. Every layer of nesting around an element's box corresponds
to another level of indentation.
Placeholder Content
After creating an outline of the elements on a page and indenting each element to
the correct level, it is helpful to place text, which will help you remember what goes
where, temporarily inside most elements.
Comments
Text notes, which the browser does not read, that are included in order to
communicate with other people who will read the code or to leave reminders and
explanations about specific elements.
