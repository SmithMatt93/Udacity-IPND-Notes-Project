Lesson 1: The Basics of the Web and HTML

Thinking Like a Programmer

There are 5 main types of thinking that are frequently used by programmers:

  1. Procedural
    Involves creating clear and concise instructions for a computer to follow.
  2. Abstract:
    Involves finding ways to improve programming efficiency by helping avoid unnecessary repetition of work.
  3. Systems
    Involves breaking down large problems into smaller problems and recognizing how those individual pieces fit together.
  4. Technological Empathy
    Involves understanding how computers work, and what they are good and bad at, and using that knowledge to your advantage while coding.
  5. Debugging
    Involves using a system to make identifying and fixing bugs quick and relatively easy.
The World Wide Web

A network of servers and other computers on which HTML documents and files are hosted and accessed.

Components of the WWW
Users:
People who use computers to access and create content on the WWW.
Computers:
Programmable machines that run programs browsers) that display pages and files found on the web.
The Internet:
The largest network of computers in the world.
Servers:
Computers that host the files that make up the web.
HTTP:
The protocol that is used to communicate between these components. This Wikipedia entry explains HyperText Transfer Protocol in extreme depth.
Webpages

Text documents, written in HTML, which are read, interpreted, and displayed by web browsers.

HTML

HyperText Markup Language is a programming language which is the basis of almost every webpage on the WWW. HTML documents consist of:

  • Text content (what you see)
  • Markup (how it looks)
  • References to other documents/files (images, video, audio, other pages, etc.
Components of HTML
Tags:
The text instructions that tell a browser how to display content. They are always contained within angled braces. They're paired together with an opening and closing tag, except for void tags.
Elements:
Units made up of a tag and the content it affects, or an opening and closing tag and the content between them.
Inline vs Block Tags

The two types of tags within HTML are inline and block tags. Block tags form a box around the contents within them, and start on a new line. Inline elements flow in between surrounding text.

Coding

The action of writing text instructions in a language that computers can understand and follow.

Correct Syntax

Computers are "stupid" because they require code to be written in a very exact way to be able to understand it. Humans are considered "smart," by contrast, because they are capable of compensating for and overcoming typos.

Lesson 2: Creating a Structured Document with HTML

Text Editors

The programs used to write and edit code for webpages. They are similar to word processors like Microsoft Word, but lack the formatting functions. Good text editors have many automatic and predictive functions that make programming easier. They can often be used to write code in multiple languages.

HTML vs CSS vs JavaScript
  • HTML is the language used to define the structure and content of a webpage.
  • CSS is the language used to define the style of the webpage.
  • JavaScript is the language used to define the interactive components of a webpage.
DOM

Document Object Model is the standard convention used in representing and interacting with elements in HTML. This means that browsers all render HTML using the same model, resulting in a common pattern in the structure of all webpages (see Branching Structure of HTML below).

Browser Developer Tools

Add-ons or built-in features that allow you to view the DOM trees that browsers interpret HTML documents as, and the styles applied to the elements contained with these HTML documents.

Branching Structure of HTML

Because HTML elements can be nested within each other, the structure of an HTML document can be analogized to a tree. The more deeply nested within other elements an individual element is, the further up and out from the trunk of the tree it would be found. For example, an image might be nested within a 'div' element which is, in turn, nested within the 'body' element. The body may be considered the trunk of the tree, the div, a bough sticking out from the trunk, and the image, a twig sticking out from the bough.

Nested Boxes and Indentation

Every single visible element of a webpage is bounded by a box and, therefore, every webpage can beken down into a series of boxes with smaller boxes inside of them (and often more boxes inside of these smaller boxes, and so on). Front end web developers do this with design mockups of webpages to begin laying out the framework of a page. Every layer of nesting around an element's box corresponds to another level of indentation.

Placeholder Content

After creating an outline of the elements on a page and indenting each element to the correct level, it is helpful to place text, which will help you remember what goes where, temporarily inside most elements.

Comments

Text notes, that the browser ignores, which are included in order to communicate with other people who will read the code (e.g. to explain why a segment of code was written in a certain way to a coworker) and to leave reminders for oneself.

Good Coding Style
  • Increases the efficiency with which one can code.
  • Makes code easier for others to read and understand.
  • Is more conducive to standardization within teams.
  • Is more visually appealing and impressive to potential employers and clients.

Lesson 3: Adding CSS Style and HTML Structure

CSS and Repetition Avoidance

One of the main benefits of CSS is that it allows programmers to avoid completing the same task repeatedly by employinig selectors. This makes it much faster and easier make changes to related elements on your page and vastly reduces the chance of errors in your code.

Selectors

Units of CSS code that define what elements of the page a particular style will be applied to. There are many different types of selectors, and they can be used in almost any combinations with each other (Major selector types below).

ID Selectors

Used to apply style to singular elements. Will override class and tag selectors.

CSS:

#id-name {
attribute: value;
 }

HTML Selected:

<div id="id-name"></div>

Class Selectors

Used to apply style to multiple elements with common purposes or structure. Generally considered the most versatile type of selector. Will override tag selectors.

CSS:

.class-name {
attribute: value;
 }

HTML Selected:

<div class="class-name" ></div>

Tag Selectors

Used to apply style to all instances of a specific type of tag.

CSS:

tagtype {
attribute: value;
 }

HTML Selected:

<tagtype></tagtype>

Attribute Selectors

Used to apply style to any type of attribute with a specific value. Has the same specificity value as a class selector.

CSS:

[attribute="value"] {
attribute: value;
 }

HTML Selected:

<div attribute="value"></div>

Positional Selectors

A family of selectors used to apply style to elements based on their positioning relative to other elements. Below is just one type.

CSS:

:nth-child(3) {
attribute: value;
 }

HTML Selected:

<div>
<div></div>
 <div></div>
 <div>3rd child element selected</div>
 </div>

Specificity

The way in which browsers decide which property values should be applied to any given elements. Each selector type has a specificity value, and when combined with other selectors, their weights are combined (the values are not actually added like decimal numbers!). The property values with higher combined specifity override those with lower.

Coding Style Tips
  • (HTML) Add one indent in front of an element for each level that it is nested within other elements.
  • Keep all code within the right side margin of the text editor to prevent the need for horizontal scrolling.
  • Include comments to help organize and navigate the code. For example, comments which catch a reader's attention ("flags") and describe the purpose of the code beneath them can be placed at the top of each major section of a stylesheet.
  • (CSS) Organize "modules" of code based on the relationship of the element(s) that it effects with the parent, siblings, and children of that element, and similar element types and classes.
